## Feature 1: Telegram Alerts to Plant Manager

### What it does

When a scenario completes and the final recommendation lands, BakeOps sends a formatted alert to a Telegram chat. During the demo, the "plant manager's" phone sits face-up on the table. Agents finish reasoning → phone buzzes → judge sees the notification. The AI just took real-world action.

### Environment Variables

Add to `backend/.env`:

```env
TELEGRAM_BOT_TOKEN=your_bot_token_here
TELEGRAM_CHAT_ID=your_chat_id_here
```

### Backend Implementation

#### Create `backend/services/telegram.py`

```python
import httpx
import os
from datetime import datetime

TELEGRAM_BOT_TOKEN = os.getenv("TELEGRAM_BOT_TOKEN")
TELEGRAM_CHAT_ID = os.getenv("TELEGRAM_CHAT_ID")
TELEGRAM_API_URL = f"https://api.telegram.org/bot{TELEGRAM_BOT_TOKEN}/sendMessage"


async def send_alert(
    scenario_title: str,
    diagnosis: str,
    recommended_action: str,
    cost_of_action: str,
    cost_of_inaction: str,
    confidence: float,
    response_time_seconds: int,
):
    """
    Send a formatted alert to the plant manager's Telegram.
    Called after the Orchestrator produces the final recommendation.
    """

    timestamp = datetime.now().strftime("%Y-%m-%d %H:%M:%S EDT")
    ref_id = f"BO-{datetime.now().strftime('%Y%m%d-%H%M%S')}"

    message = (
        f"🚨 *BAKEOPS ALERT*\n"
        f"━━━━━━━━━━━━━━━━━━\n"
        f"*{scenario_title}*\n\n"
        f"📋 *Diagnosis:*\n"
        f"{diagnosis}\n\n"
        f"✅ *Recommended Action:*\n"
        f"{recommended_action}\n\n"
        f"💰 *Cost Analysis:*\n"
        f"  • Intervention: {cost_of_action}\n"
        f"  • If unresolved: {cost_of_inaction}\n\n"
        f"📊 *Confidence:* {int(confidence * 100)}%\n"
        f"⏱ *Response Time:* {response_time_seconds}s\n\n"
        f"━━━━━━━━━━━━━━━━━━\n"
        f"Ref: {ref_id}\n"
        f"Plant: North York, ON\n"
        f"Time: {timestamp}\n"
        f"_Generated by BakeOps Command Center_"
    )

    try:
        async with httpx.AsyncClient() as client:
            response = await client.post(
                TELEGRAM_API_URL,
                json={
                    "chat_id": TELEGRAM_CHAT_ID,
                    "text": message,
                    "parse_mode": "Markdown",
                },
                timeout=5.0,
            )
            if response.status_code == 200:
                print(f"[Telegram] Alert sent: {ref_id}")
            else:
                print(f"[Telegram] Failed: {response.text}")
    except Exception as e:
        # CRITICAL: Never let Telegram failure break the demo
        print(f"[Telegram] Error (non-fatal): {e}")
```

## Feature 2: Live Webcam Label Scanning

### What it does

Instead of uploading a photo, the judge holds up ANY food package to your laptop's webcam. A live video preview shows on the dashboard. They click "Scan" (or it auto-captures after holding still). The frame is sent to Claude Vision, which reads the ingredient list. Recipe Chemist agent activates automatically and begins reformulating. No file picker, no uploads, no typing. Judge holds up a granola bar → AI reformulates it live.

This is the most memorable physical interaction in your demo. Every other team's demo is "type something." Yours is "hold up a snack."

### Architecture

```
Webcam → Browser MediaStream → Canvas capture → Base64 frame
    → POST /api/scan-label → Claude Vision reads ingredients
    → Recipe Chemist agent activates → SSE streams reasoning
    → Dashboard shows reformulation in real time
```

### Frontend Implementation

#### Create `frontend/src/components/WebcamScanner.tsx`

This component provides:
1. A button in the CommandBar area to open the scanner
2. A modal/overlay showing the live webcam feed
3. A "Scan" button to capture a frame
4. Auto-close after capture and send to backend

```tsx
// Component structure (implement with these behaviors):

// State:
//   isOpen: boolean (modal visibility)
//   isScanning: boolean (waiting for backend response)
//   videoRef: ref to <video> element
//   canvasRef: ref to hidden <canvas> element (for frame capture)

// On open:
//   Request webcam access: navigator.mediaDevices.getUserMedia({ video: { facingMode: 'environment', width: 1280, height: 720 } })
//   Attach stream to videoRef

// On "Scan" button click:
//   Draw current video frame to hidden canvas
//   Convert canvas to base64: canvas.toDataURL('image/jpeg', 0.85)
//   Strip the data URL prefix to get raw base64
//   POST to /api/scan-label with { image: base64string }
//   Close modal
//   The backend response triggers Recipe Chemist via existing SSE stream

// On close:
//   Stop all media tracks: stream.getTracks().forEach(track => track.stop())

// UI layout of the modal:
//   Dark overlay (bg-black/80)
//   Center: video feed (rounded-lg, max-width 640px)
//   Below video: "Hold the ingredient label steady in front of the camera"
//   Bottom: [Cancel] button (left) and [Scan Label] button (right, amber accent)
//   When scanning: show a brief scanning animation overlay on the video
```

#### Key implementation details for the webcam

```typescript
// Opening webcam
const openCamera = async () => {
  try {
    const stream = await navigator.mediaDevices.getUserMedia({
      video: {
        facingMode: 'environment',  // prefer back camera on mobile, works fine on laptop
        width: { ideal: 1280 },
        height: { ideal: 720 },
      }
    });
    if (videoRef.current) {
      videoRef.current.srcObject = stream;
      videoRef.current.play();
    }
  } catch (err) {
    console.error('Camera access denied:', err);
    // Show fallback: file upload input
  }
};

// Capturing a frame
const captureFrame = (): string | null => {
  const video = videoRef.current;
  const canvas = canvasRef.current;
  if (!video || !canvas) return null;

  canvas.width = video.videoWidth;
  canvas.height = video.videoHeight;
  const ctx = canvas.getContext('2d');
  if (!ctx) return null;

  ctx.drawImage(video, 0, 0);
  const dataUrl = canvas.toDataURL('image/jpeg', 0.85);
  return dataUrl.split(',')[1]; // strip "data:image/jpeg;base64," prefix
};

// Sending to backend
const scanLabel = async () => {
  const base64 = captureFrame();
  if (!base64) return;

  setIsScanning(true);
  closeCamera();

  // This POST triggers the scenario via existing SSE mechanism
  const response = await fetch('/api/scan-label', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ image: base64 }),
  });
  // The actual agent response comes via the existing SSE stream
  // No need to handle the POST response body
};

// Cleanup
const closeCamera = () => {
  const stream = videoRef.current?.srcObject as MediaStream;
  stream?.getTracks().forEach(track => track.stop());
  setIsOpen(false);
};
```

### Backend Implementation

#### Add endpoint `POST /api/scan-label`

```python
@app.post("/api/scan-label")
async def scan_label(request: Request):
    """
    Receive a base64-encoded image of a food label.
    Send to Claude Vision to extract ingredients.
    Trigger Recipe Chemist agent with the extracted data.
    Returns SSE stream (same as /api/trigger).
    """
    body = await request.json()
    image_base64 = body.get("image")

    if not image_base64:
        return JSONResponse({"error": "No image provided"}, status_code=400)

    # This endpoint returns an SSE stream, same as /api/trigger
    return EventSourceResponse(
        process_label_scan(image_base64)
    )
```

#### Create label scanning flow

```python
async def process_label_scan(image_base64: str):
    """
    1. Send image to Claude Vision to extract ingredient list
    2. Pass extracted ingredients to Recipe Chemist agent
    3. Stream everything via SSE events
    """

    # Step 1: Emit "scanning" event
    yield create_sse_event(
        event_type="label_scan_started",
        agent_id="orchestrator",
        data={"status": "Analyzing label with computer vision..."}
    )

    # Step 2: Call Claude Vision to extract ingredients from the image
    import anthropic
    client = anthropic.Anthropic()

    vision_response = client.messages.create(
        model="claude-sonnet-4-20250514",
        max_tokens=1500,
        messages=[
            {
                "role": "user",
                "content": [
                    {
                        "type": "image",
                        "source": {
                            "type": "base64",
                            "media_type": "image/jpeg",
                            "data": image_base64,
                        }
                    },
                    {
                        "type": "text",
                        "text": (
                            "Read this food product label. Extract:\n"
                            "1. The product name\n"
                            "2. The complete ingredient list exactly as written\n"
                            "3. Any allergen warnings\n"
                            "4. The brand name if visible\n\n"
                            "Return ONLY a JSON object with keys: "
                            "product_name, brand, ingredients (as a single string), allergens (as a list). "
                            "No markdown, no explanation, just the JSON."
                        )
                    }
                ]
            }
        ]
    )

    # Parse the extracted ingredients
    import json
    try:
        extracted = json.loads(vision_response.content[0].text)
    except json.JSONDecodeError:
        # Fallback: treat the whole response as ingredient text
        extracted = {
            "product_name": "Unknown Product",
            "brand": "Unknown",
            "ingredients": vision_response.content[0].text,
            "allergens": []
        }

    # Step 3: Emit extraction result
    yield create_sse_event(
        event_type="label_scan_result",
        agent_id="orchestrator",
        data={
            "product_name": extracted.get("product_name", "Unknown"),
            "brand": extracted.get("brand", "Unknown"),
            "ingredients": extracted.get("ingredients", ""),
            "allergens": extracted.get("allergens", []),
        }
    )

    # Step 4: Build scenario text from extracted data and trigger Recipe Chemist
    scenario_text = (
        f"Clean label reformulation request for: {extracted.get('product_name', 'scanned product')} "
        f"by {extracted.get('brand', 'unknown brand')}. "
        f"Current ingredients: {extracted.get('ingredients', 'unknown')}. "
        f"Identify all synthetic, artificial, or highly processed ingredients. "
        f"Propose natural clean-label replacements with cost and functional impact."
    )

    # Step 5: Run through the normal orchestrator → recipe chemist flow
    # Reuse your existing scenario processing pipeline
    async for event in run_scenario(scenario_text, force_agent="recipe_chemist"):
        yield event
```

#### Important: the `run_scenario` function

The `process_label_scan` function needs to call into your existing scenario pipeline. Depending on how your LangGraph is structured, you either:

**Option A:** Call the orchestrator with a hint to route to Recipe Chemist:
```python
async for event in run_scenario(scenario_text, force_agent="recipe_chemist"):
    yield event
```

**Option B:** Call the Recipe Chemist agent directly (simpler, recommended for this feature):
```python
async for event in run_recipe_chemist(scenario_text):
    yield event
```

Choose whichever integrates cleanly with your existing code. The important thing is that the Recipe Chemist agent receives the extracted ingredients and produces the reformulation analysis, streaming via the same SSE event format the frontend already handles.

### Frontend: Handling the scan response

The frontend needs to handle two new SSE event types:

**`label_scan_started`:** Show in the ActiveStream panel:
- Orchestrator card activates
- Stream text: "Analyzing label with computer vision..."
- Show the captured image as a small thumbnail (store the base64 in local state when capturing)

**`label_scan_result`:** Show the extracted product info:
- Product name in Fraunces 16px
- Brand in mono 11px
- Ingredient list in sans 13px (this is what was read from the label)
- Then Recipe Chemist activates and the normal agent streaming begins

The transition should feel seamless: scan → "reading label..." → product identified → Recipe Chemist starts reformulating → recommendation card appears → Telegram alert sends.

### Fallback: if webcam doesn't work
